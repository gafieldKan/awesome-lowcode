## 1. 平台简介

### 1.1 什么是低代码平台

低代码平台（Low-Code Platform）是一种可视化的开发环境，让开发者无需编写大量代码，即可通过拖拽、配置等简单操作快速构建应用程序。

### 1.2 平台特性

**核心功能**
- ✅ 拖拽式页面构建器
- ✅ 25+ 种字段类型的数据模型
- ✅ 可视化工作流编排（6 种触发器、12 种操作）
- ✅ 完整 RBAC 权限系统
- ✅ 插件化架构（支持自定义扩展）
- ✅ 事件驱动架构（钩子、拦截器）

**高级功能**
- ✅ 多租户支持（租户隔离、配额管理）
- ✅ 插件市场（安装/卸载/更新）
- ✅ 性能分析工具
- ✅ 调试工具面板
- ✅ CLI 命令行工具

**技术特性**
- ✅ 元数据驱动，支持热更新
- ✅ JSON Schema 导出/导入
- ✅ 前端组件库（Ant Design 6）
- ✅ 状态管理（Zustand）

## 4. 表单设计器

### 4.1 功能概述

表单设计器用于创建和编辑可视化表单，支持拖拽组件、配置属性、管理布局。

### 4.2 界面组成

| 区域 | 功能说明 |
|------|----------|
| 组件面板 | 显示所有可用组件，支持拖拽 |
| 画布区域 | 放置和排列组件的区域 |
| 属性面板 | 配置选中组件的属性 |
| 组件树 | 显示组件层级结构 |

### 4.3 操作步骤

#### 添加组件

1. 从左侧组件面板选择组件
2. 按住鼠标拖拽到画布区域
3. 松开鼠标完成添加

#### 配置组件属性

1. 在画布中点击要编辑的组件
2. 右侧属性面板自动展开
3. 修改属性值（如文本、占位符、类型等）
4. 修改实时生效

#### 删除组件

1. 在画布中选中组件
2. 点击工具栏"删除选中"按钮

### 4.4 使用技巧

- **快速选择**: 在组件树中点击可快速选中组件
- **JSON 导出**: 点击导出按钮可保存为 JSON Schema
- **清空画布**: 一键清除所有组件重新开始

## 5. 模型设计器

### 5.1 功能概述

模型设计器用于定义数据结构，支持 25+ 种字段类型，自动生成对应的表单和 CRUD 操作。

### 5.2 创建数据模型

1. 点击"新建模型"按钮
2. 输入模型名称（如：User、Product）
3. 填写描述信息（可选）
4. 点击"创建"完成

### 5.3 添加字段

1. 在模型列表中选择要编辑的模型
2. 点击"添加字段"按钮
3. 配置字段信息

### 5.4 字段类型说明

| 类型分类 | 字段类型 |
|----------|----------|
| 基础类型 | string, text, richText, number, integer, float, boolean |
| 日期时间 | date, datetime, time |
| 选择类型 | select, multiSelect, radio, checkbox, tag |
| 文件类型 | image, file, images, files |
| 高级类型 | json, code, color, rating, percent, currency, phone, email, url, address, formula, autoNumber |

### 5.5 验证规则

| 规则 | 说明 | 示例 |
|------|------|------|
| required | 必填 | `{ type: 'required' }` |
| pattern | 正则匹配 | `{ type: 'pattern', pattern: '^[a-z]+$' }` |
| min/max | 最小/最大值 | `{ type: 'min', value: 0 }` |
| minLength/maxLength | 长度限制 | `{ type: 'minLength', value: 3 }` |
| email | 邮箱格式 | `{ type: 'email' }` |
| phone | 手机号格式 | `{ type: 'phone' }` |
| url | URL 格式 | `{ type: 'url' }` |
| custom | 自定义验证 | `{ type: 'custom', validate: fn }` |

### 5.6 自动生成表单

1. 切换到"表单生成"标签页
2. 选择已创建的模型
3. 系统自动生成对应表单
4. 可实时查看和测试表单数据

## 6. 工作流设计器

### 6.1 功能概述

工作流设计器用于创建自动化业务流程，支持可视化编排、触发器配置和动作管理。

### 6.2 触发器类型（6 种）

| 触发器 | 说明 | 使用场景 |
|--------|------|----------|
| 手动触发 | 用户手动点击执行 | 临时任务、测试 |
| 定时触发 | 按指定时间执行 | 日报生成、定时备份 |
| Webhook | HTTP 请求触发 | 第三方系统集成 |
| 数据变更 | 数据变化时触发 | 库存预警、状态同步 |
| 创建数据 | 创建数据时触发 | 新用户欢迎 |
| 删除数据 | 删除数据时触发 | 数据清理通知 |

### 6.3 动作类型（12 种）

| 动作类型 | 说明 |
|----------|------|
| 创建数据 | 向指定模型添加新数据 |
| 更新数据 | 修改现有数据记录 |
| 删除数据 | 删除指定数据 |
| 发送邮件 | 发送电子邮件通知 |
| 发送短信 | 发送短信通知 |
| HTTP 请求 | 调用外部 API 接口 |
| 执行脚本 | 运行自定义脚本 |
| 通知用户 | 发送站内通知 |
| 审批流程 | 发起审批请求 |
| 条件判断 | 根据条件分支执行 |
| 延时等待 | 延迟执行后续操作 |
| 循环操作 | 批量处理数据 |

### 6.4 使用示例：新用户注册流程

```
触发器：创建数据 (User 模型)
  ↓
动作 1: 发送邮件 (发送欢迎邮件)
  ↓
动作 2: 创建数据 (创建用户档案)
  ↓
动作 3: 通知用户 (发送站内消息)
```

| panel | 面板插件 | 侧边栏面板 |

### 7.2 插件生命周期

| 生命周期 | 说明 | 调用时机 |
|----------|------|----------|
| init | 初始化 | 插件注册时 |
| onLoad | 加载 | 插件加载时 |
| onSave | 保存 | 插件保存时 |
| onDestroy | 销毁 | 插件销毁时 |
| onActivate | 激活 | 插件激活时 |
| onDeactivate | 停用 | 插件停用时 |

### 7.3 注册插件

```js
import { registerPlugin, PluginType } from 'lowcode/core/PluginRegistry'

registerPlugin({
  name: 'my-custom-input',
  type: PluginType.FORM_COMPONENT,
  version: '1.0.0',
  description: '自定义输入组件',

  async init() {
    console.log('插件初始化')
  },

  async onLoad(context) {
    console.log('插件加载', context)
  },

  async onSave(data) {
    console.log('插件保存', data)
  },

  async onDestroy() {
    console.log('插件销毁')
  },
})
```

### 7.4 插件市场

import { pluginMarket } from 'lowcode/plugins/PluginMarket'

// 获取已安装的插件

## 8. 权限管理

### 8.1 权限级别

| 级别 | 说明 |
|------|------|
| none | 无权限 |
| read | 只读 |
| write | 读写 |
| admin | 管理员 |
| full | 完全控制 |

### 8.2 操作类型（14 种）

```js
Operations = {
  READ: 'read',       WRITE: 'write',     CREATE: 'create',
  DELETE: 'delete',   UPDATE: 'update',   SUBMIT: 'submit',
  CANCEL: 'cancel',   APPROVE: 'approve', REJECT: 'reject',
  SHARE: 'share',     EXPORT: 'export',   IMPORT: 'import',
  PRINT: 'print',     FULL: 'full'
}
```

### 8.3 数据范围

| 范围 | 说明 |
|------|------|
| ALL | 全部数据 |
| MINE | 仅本人数据 |
| DEPT | 本部门数据 |
| DEPT_AND_SUB | 本部门及子部门数据 |
| CUSTOM | 自定义范围 |

### 8.4 设置权限

```js
import { permissionManager, Operations, DataScope } from 'lowcode/core/PermissionManager'

// 1. 分配角色
permissionManager.assignRole('user123', 'manager')

// 2. 设置模型权限
permissionManager.setModelPermission('order', 'manager', [
  Operations.READ,
  Operations.WRITE,
  Operations.APPROVE,
])

// 3. 设置数据范围
permissionManager.setDataScope('order', 'manager', DataScope.DEPT)

// 4. 设置字段权限
permissionManager.setFieldPermission('user', 'salary', 'editor', {
  visible: false,
  readable: false,
  writable: false,
})

// 5. 检查权限
const canApprove = permissionManager.hasModelPermission(
  'order',
  'manager',
  Operations.APPROVE
)
```

## 9. 事件系统

### 9.1 事件订阅

```js
import { eventEngine, EventPriority } from 'lowcode/engine/EventEngine'

// 订阅事件
eventEngine.on('user:login', (user) => {
  console.log('用户登录:', user.name)
})

// 带优先级的订阅
eventEngine.on('order:submit', handler, EventPriority.HIGH)

// 一次性订阅
eventEngine.once('data:load', handler)

// 取消订阅
eventEngine.off('user:login', handlerId)
```

### 9.2 事件拦截器

```js
import { addInterceptor } from 'lowcode/engine/EventEngine'

// 添加拦截器，返回 false 可阻止后续执行
addInterceptor('data:delete', (data) => {
  if (!canDelete) {
    return false // 阻止删除
  }
})
```

### 9.3 前后钩子

```js
// 添加前置钩子
eventEngine.addHook('app:start', async () => {
  await loadConfig()
}, 'before')

// 添加后置钩子
eventEngine.addHook('app:start', async () => {
  await initPlugins()
}, 'after')
```

## 12. 常见问题

### 12.1 安装问题

**Q: npm install 失败**

```bash
# 尝试清理缓存后重新安装
npm cache clean --force
npm install

# 或使用国内镜像
npm install --registry=https://registry.npmmirror.com
```

### 12.2 使用问题

**Q: 组件拖拽无反应**
- 检查浏览器兼容性（推荐使用 Chrome/Edge）
- 清除浏览器缓存
- 确认 JavaScript 已启用

### 12.3 开发相关

**Q: 如何开发自定义插件？**

查看 [docs/二次开发指南.md](docs/二次开发指南.md)

**Q: 如何进行代码检查？**

```bash
npm run lint    # ESLint 检查
npm run format  # Prettier 格式化
npm run test    # 运行测试
```
