## 1. 平台简介

### 1.1 什么是低代码平台

低代码平台（Low-Code Platform）是一种可视化的开发环境，让开发者无需编写大量代码，即可通过拖拽、配置等简单操作快速构建应用程序。

### 1.2 平台特性

- **拖拽式编辑** - 可视化组件拖拽，所见即所得
- **组件丰富** - 内置输入框、按钮、表格等常用组件
- **模型驱动** - 数据模型自动生成表单
- **工作流编排** - 可视化流程编排，自动化业务逻辑
- **实时预览** - 修改即时生效，无需刷新
- **持久化存储** - 设计自动保存，支持导出导入

### 1.3 技术架构

```
┌─────────────────────────────────────────┐
│           用户界面层                      │
│  ┌─────────┬─────────┬─────────┐        │
│  │表单设计器│ 模型设计器│ 工作流设计器│        │
│  └─────────┴─────────┴─────────┘        │
├─────────────────────────────────────────┤
│           核心引擎层                      │
│  ┌─────────────────────────────────┐    │
│  │  状态管理 (Zustand) + 持久化     │    │
│  └─────────────────────────────────┘    │
├─────────────────────────────────────────┤
│           数据服务层                      │

│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

## 2. 快速开始

### 2.1 环境要求

- Node.js >= 18.0
- npm >= 9.0
- 现代浏览器（Chrome、Edge、Firefox）

### 2.2 安装步骤

```bash
# 1. 克隆项目
git clone <项目地址>
cd awesome-lowcode

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev

# 4. 构建生产版本
npm run build
```

### 2.3 访问平台

启动后，在浏览器中访问：

- **首页**: `http://localhost:3000/`
- **表单设计器**: `http://localhost:3000/editor`

## 3. 表单设计器

### 3.1 功能概述

表单设计器用于创建和编辑可视化表单，支持拖拽组件、配置属性、管理布局。

### 3.2 界面组成

| 区域 | 功能说明 |
|------|----------|
| 组件面板 | 显示所有可用组件，支持拖拽 |
| 画布区域 | 放置和排列组件的区域 |
| 属性面板 | 配置选中组件的属性 |
| 组件树 | 显示组件层级结构 |

### 3.3 操作步骤

#### 添加组件

**方法一：拖拽添加**
1. 从左侧组件面板选择组件
2. 按住鼠标拖拽到画布区域
3. 松开鼠标完成添加

**方法二：点击添加**
1. 找到要添加的组件
2. 点击组件旁的"添加"按钮
3. 组件自动添加到画布

#### 配置组件属性

1. 在画布中点击要编辑的组件
2. 右侧属性面板自动展开
3. 修改属性值（如文本、占位符、类型等）
4. 修改实时生效

#### 组件属性说明

| 组件类型 | 可配置属性 |
|----------|------------|
| 输入框 | 占位符、是否必填、默认值 |
| 按钮 | 按钮文本、类型（主要/默认/危险） |
| 下拉框 | 选项列表、默认选项 |
| 表格 | 列定义、数据源 |

#### 删除组件

1. 在画布中选中组件
2. 点击工具栏"删除选中"按钮
3. 或在组件树中点击删除图标

#### 清空画布

点击工具栏"清空画布"按钮，确认后即可清空所有组件。

### 3.4 使用技巧

- **快速选择**: 在组件树中点击可快速选中组件
- **批量操作**: 支持多选组件统一配置属性
- **撤销重做**: 使用 Ctrl+Z / Ctrl+Y 撤销上一步操作
- **JSON 导出**: 点击导出按钮可保存为 JSON Schema

## 4. 模型设计器

### 4.1 功能概述

模型设计器用于定义数据结构，自动生成对应的表单和 CRUD 操作。

### 4.2 创建数据模型

1. 点击"新建模型"按钮
2. 输入模型名称（如：User、Product）
3. 填写描述信息（可选）
4. 点击"创建"完成

### 4.3 添加字段

1. 在模型列表中选择要编辑的模型
2. 点击"添加字段"按钮
3. 配置字段信息：
   - 字段名
   - 字段类型
   - 是否必填
   - 默认值

### 4.4 字段类型说明

| 类型 | 说明 | 示例值 |
|------|------|--------|
| string | 文本 | "Hello" |
| number | 数字 | 123 |
| boolean | 布尔值 | true/false |
| date | 日期 | "2026-04-19" |
| email | 邮箱 | user@example.com |
| phone | 电话 | "13800138000" |
| url | 链接 | "https://example.com" |
| json | JSON 对象 | {"key": "value"} |

### 4.5 字段属性

| 属性 | 说明 |
|------|------|
| 必填 | 该字段不能为空 |
| 唯一 | 该字段值在数据集中唯一 |
| 索引 | 为该字段创建索引，加速查询 |

### 4.6 自动生成表单

1. 切换到"表单生成"标签页
2. 选择已创建的模型
3. 系统自动生成对应表单
4. 可实时查看和测试表单数据

### 4.7 导出 Schema

点击"导出 Schema"按钮，可将当前模型结构导出为 JSON 格式，用于：
- 数据库建表脚本
- API 接口文档
- 前后端数据结构对齐

## 5. 工作流设计器

### 5.1 功能概述

工作流设计器用于创建自动化业务流程，支持触发器配置和动作编排。

### 5.2 界面组成

| 区域 | 功能说明 |
|------|----------|
| 工作流列表 | 显示已创建的工作流 |
| 画布区域 | 配置触发器和动作序列 |
| 属性面板 | 编辑工作流基本信息 |

### 5.3 创建工作流

1. 点击"新建"按钮
2. 输入工作流名称（如：订单处理流程）
3. 填写描述信息
4. 点击"创建"完成

### 5.4 配置触发器

触发器是工作流执行的起点，支持的类型：

| 触发器类型 | 说明 | 使用场景 |
|------------|------|----------|
| 手动触发 | 用户手动点击执行 | 临时任务、测试 |
| 定时触发 | 按指定时间执行 | 日报生成、定时备份 |
| Webhook | HTTP 请求触发 | 第三方系统集成 |
| 数据变更 | 数据变化时触发 | 库存预警、状态同步 |
| 创建数据 | 创建数据时触发 | 新用户欢迎 |
| 删除数据 | 删除数据时触发 | 数据清理、通知 |

### 5.5 添加动作

动作是工作流执行的具体操作：

| 动作类型 | 说明 |
|----------|------|
| 创建数据 | 向指定模型添加新数据 |
| 更新数据 | 修改现有数据记录 |
| 删除数据 | 删除指定数据 |
| 发送邮件 | 发送电子邮件通知 |
| 发送短信 | 发送短信通知 |
| HTTP 请求 | 调用外部 API 接口 |
| 执行脚本 | 运行自定义脚本 |
| 通知用户 | 发送站内通知 |
| 审批流程 | 发起审批请求 |
| 条件判断 | 根据条件分支执行 |
| 延时等待 | 延迟执行后续操作 |
| 循环操作 | 批量处理数据 |

### 5.6 添加动作步骤

1. 在工作流画布中找到"操作"区域
2. 从下拉菜单选择动作类型
3. 配置动作参数
4. 点击"保存"完成添加

### 5.7 执行工作流

1. 确保工作流状态为"已启用"
2. 点击"运行"按钮
3. 查看执行日志和结果

### 5.8 使用示例

#### 示例：新用户注册流程

```
触发器：创建数据 (User 模型)
        ↓
动作 1: 发送邮件 (发送欢迎邮件)
        ↓
动作 2: 创建数据 (创建用户档案)
        ↓
动作 3: 通知用户 (发送站内消息)
```

#### 示例：订单处理流程

```
触发器：创建数据 (Order 模型)
        ↓
动作 1: 条件判断 (订单金额 > 1000?)
        ├─ 是 → 动作 2: 通知管理员审核
        └─ 否 → 动作 3: 自动确认订单
              ↓
        动作 4: 发送邮件 (订单确认)
```

## 6. 常见问题

### 6.1 安装问题

**Q: npm install 失败**
```bash
# 尝试清理缓存后重新安装
npm cache clean --force
npm install

# 或使用国内镜像
npm install --registry=https://registry.npmmirror.com
```

**Q: 开发服务器无法启动**
```bash
# 检查端口是否被占用
lsof -i :3000

# 修改端口启动
PORT=3001 npm run dev
```

### 6.2 使用问题

**Q: 组件拖拽无反应**
- 检查浏览器兼容性（推荐使用 Chrome/Edge）
- 清除浏览器缓存
- 确认 JavaScript 已启用

**Q: 属性面板不显示**
- 确保已在画布中选中组件
- 检查组件树中是否有选中状态
- 刷新页面重试

**Q: 数据持久化失效**
- 检查浏览器是否允许本地存储
- 确认未使用隐私模式/无痕模式
- 尝试手动导出保存

### 6.3 导出导入

**导出设计**
1. 点击工具栏"导出"按钮
2. 保存 JSON 文件到本地

**导入设计**
1. 点击"导入"按钮
2. 选择之前导出的 JSON 文件
3. 确认后设计自动加载

### 6.4 性能优化

- 大型表单建议分组管理
- 减少单页面组件数量（建议 < 50 个）
- 定期清理不用的设计稿
- 生产环境使用 build 构建

### 6.5 技术支持

如遇到其他问题，请：
1. 查看控制台错误信息
2. 检查浏览器兼容性
3. 查阅项目 README.md
4. 提交 Issue 反馈
