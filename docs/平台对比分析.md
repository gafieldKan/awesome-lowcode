# 低代码平台对比分析

## 1. 成熟低代码平台分析

### 1.1 Frappe Framework

**核心特性：**
- 完整的元数据驱动架构
- 强大的 DocType 系统（数据模型）
- 内置工作流引擎
- 完善的权限系统（基于角色的访问控制）
- 服务器脚本和客户端脚本
- 自动 REST API 生成
- 支持多租户

**架构特点：**
```
Frappe/
├── frappe/           # 核心框架
├── core/             # 核心模块
├── website/          # 网站模块
├── workflow/         # 工作流模块
└── [custom apps]/   # 自定义应用
```

**值得借鉴：**
1. DocType 作为一切数据的基础
2. 自动化的 CRUD API
3. 服务器/客户端脚本钩子
4. 细粒度权限控制

### 1.2 NocoBase

**核心特性：**
- 数据模型驱动
- 插件化架构
- 可视化数据建模
- 灵活的角色权限
- 工作流自动化
- 支持多数据源

**架构特点：**
```
NocoBase/
├── packages/
│   ├── core/
│   │   ├── client/
│   │   ├── database/
│   │   ├── server/
│   ├── plugins/
│   │   ├── users/
│   │   ├── workflow/
│   │   └── [其他插件]
```

**值得借鉴：**
1. 完全插件化设计
2. 数据库与 UI 分离
3. 统一的插件管理

### 1.3 Appsmith

**核心特性：**
- 可视化 UI 构建器
- 丰富的组件库
- 数据源集成（API、数据库）
- JavaScript 表达式
- 版本控制
- 协作编辑

**值得借鉴：**
1. 强大的数据绑定系统
2. JavaScript 表达式引擎
3. 组件属性面板设计

### 1.4 阿里云低代码引擎

**核心特性：**
- 物料市场
- 页面编辑器
- 逻辑编排
- 应用发布

**值得借鉴：**
1. 物料体系（组件、区块、模板）
2. Schema 标准化
3. 扩展点设计

## 2. Awesome Lowcode 改进方向

### 2.1 核心架构改进

| 当前状态 | 目标状态 | 优先级 |
|----------|----------|--------|
| 基础插件系统 | 完整插件生命周期管理 | 高 |
| 简单 Schema 管理 | 完整元数据驱动 | 高 |
| 基础权限控制 | RBAC + 数据权限 | 高 |
| 无事件系统 | 完整事件总线 | 高 |
| 简单工作流 | 可视化流程编排 | 中 |

### 2.2 新增核心能力

1. **应用引擎 (App Engine)**
   - 应用生命周期管理
   - 路由管理
   - 状态管理

2. **渲染引擎 (Rendering Engine)**
   - 组件渲染
   - 属性绑定
   - 事件处理

3. **事件引擎 (Event Engine)**
   - 全局事件总线
   - 事件订阅/发布
   - 事件拦截器

4. **表达式引擎 (Expression Engine)**
   - 变量解析
   - 函数计算
   - 条件判断

### 2.3 数据模型增强

| 特性 | 当前 | 目标 |
|------|------|------|
| 字段类型 | 基础类型 | 支持 15+ 类型 |
| 关联关系 | 无 | 支持关联/聚合 |
| 索引 | 标记 | 实际索引支持 |
| 验证规则 | 基础 | 完整验证规则 |
| 计算字段 | 无 | 支持公式计算 |
| 字段历史 | 无 | 版本追踪 |

### 2.4 权限系统增强

| 权限类型 | 当前 | 目标 |
|----------|------|------|
| 角色管理 | 基础 | 完整 RBAC |
| 模型权限 | 基础 | 支持 8 种操作 |
| 字段权限 | 基础 | 可见/可读/可写/必填 |
| 数据权限 | 无 | 支持数据范围 |
| 菜单权限 | 基础 | 完整菜单权限 |
| 操作权限 | 基础 | 按钮级权限 |
